h1. Ayuda para el Despliegue Manual

El despliegue manual es una naturaleza de pase donde los elementos que se despliegan
no están automatizados y requieren que un usuario responsable lo realice a mano.

Ejemplos de despliegue manual:

|_. Tipo    |_. Acción |
| Definiciones (DDL) de tablas de BB.DD. | Un usuario deberá ejecutar cuidadosamente los scripts a mano |
| Ejecutables distribuidos a puestos cliente | Un usuario se encargará de crear el paquete de distribución con herramientas tipo Microsoft SMS® |
| Documentación enviada a usuario | Mediante el correo de pase manual, un usuario final (ej. Dep. de Calidad) podrá acceder a un documento generado como parte del ciclo de vida del software |

h2. ID de Acción

Poner un identificador (nombre corto), que se utilizará para denominar la acción de usuario que se asignará a los roles asignados. 

Por ejemplo, si ponemos ¿oracle¿, al guardar el formulario se convertirá en ¿action.manualdeploy.role.oracle¿ automáticamente.  

h2. Nombre de la actividad

Una pequeña descripción que se utilizará en el log de pase, petición y en los correos enviados. 

h2. Rutas

Listado de rutas (o cualquier parte del nombre del fichero) en expresión regular que se comparará contra los elementos del pase para activar esta entrada de pase manual.

h2. Roles

Roles a los que se aplicará la acción seleccionada.

h2. Descripción

Una descripción larga de la actividad, donde podremos documentar el proceso a realizar manualmente. Este texto se enviará en el correo de la petición de pase manual como ¿observaciones¿. 

El texto está en formato Textile, permitiendo utilizar etiquetas que luego se convierten a HTML, como por ejemplo

*	h1., h2., h3., h4.  ¿ Heading 1, 2, 3, 4
*	Asteriscos ¿*¿ ¿ como puntos en un listado
*	Tablas (con ¿|¿) 

Consultar manuales de Textile en internet para una descripción completa, "por ejemplo aquí":http://redcloth.org/hobix.com/textile/quick.html hay un buen manual.
